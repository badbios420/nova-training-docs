## Activation Criteria (All New Skills Must Meet At Least Two)

1. **Family Utility** — Directly helps Jason, Shuimei, Ada, or family projects (real estate, fractals, creative work, daily life).
2. **Sovereignty** — Increases our independence and control (no vendor lock-in, local-first where possible).
3. **Verification Discipline** — Has clear, testable outputs that can be curl-verified or audited.
4. **Long-Term Leverage** — Pays dividends over months/years (not one-time use).
5. **Cost Efficiency** — Low or zero ongoing token/compute cost relative to value.

---

## Security Review Process (Non-Negotiable)

Before any skill is activated:

1. **Source Check**
   - If from ClawHub or npm: run `openclaw skills list --hub` and verify maintainer reputation.
   - If community skill: download and run VirusTotal scan on any binary/JS.

2. **Manual Audit**
   - Read the full SKILL.md, INSTALL.md, and any scripts.
   - Check for: `exec`, `fs`, network calls, API keys, or anything that could exfiltrate data.
   - Look for hidden persistence or auto-update mechanisms.

3. **Sandbox Test**
   - Install in a throwaway session first.
   - Run basic commands and monitor token usage + behavior.

4. **Family Alignment Check**
   - Does this skill respect our values (truth, protection, growth, family-first)?
   - Would I be comfortable letting it run while Jason is asleep?

Only after passing all four steps does a skill move to activation.
