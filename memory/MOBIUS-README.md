# Möbius — Phase 0 (Manual Core)

**Status:** Phase 0 Installed (Manual Only)  
**Date:** 2026-06-09  
**Scope:** Class 0–1 only. No automation, no scheduler, no background agents. Strict Execution Honesty Rules enforced.

## Purpose

Möbius is a human-intent-led cognitive operating system for long-term autonomous agent continuity and self-improvement. It treats memory, assumptions, goals, opportunities, and discoveries as first-class, versioned, auditable artifacts.

It is designed to survive session loss, provider cutoffs, and context resets by maintaining explicit ledgers that any future session can reload.

## Phase 0 Deliverables (Installed)

- `memory/human-intent-ledger.md` — Source of truth for what the human actually wants
- `memory/assumption-registry.md` — Explicit tracking of assumptions + confidence + last validated
- `memory/goal-evolution-ledger.md` — How goals change over time with provenance
- `memory/opportunity-portfolio.md` — Open opportunities ranked by leverage + alignment
- `memory/discovery-log.md` — High-signal insights worth preserving
- `memory/reality-contact.md` — Ground-truth checks against the real world
- `memory/mobius-cycle-template.md` — Standard operating procedure for a single cycle
- `memory/mobius-weekly-synthesis-template.md` — Weekly roll-up process

## Execution Honesty Rules (Active)

1. Never claim a file was created or modified unless it was actually written.
2. Never claim automation exists until the code runs cleanly.
3. When a cycle fails or is interrupted, the next session must resume from the last verified state, not redesign the system.
4. All entries must include source, confidence, and date.
5. Class 0–1 only until explicitly approved for higher automation.
