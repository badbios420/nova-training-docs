# Research Candidate: Verification Checklist for Self-Improvement

**Status:** Candidate (not promoted)  
**Date:** 2026-06-10  
**Source:** Research synthesis + GPT feedback (2026-06-10)  
**Trigger:** Post-Reality Audit (2026-06-09)

## Core Principle
Generation ≠ Verification

Every self-improvement or architecture change workflow must end with an explicit verification gate.

## Proposed Checklist (to be appended to relevant workflows)

- [ ] Was the change actually created / written?
- [ ] Was it committed to the repo?
- [ ] Was it pushed?
- [ ] Was the result independently verified (not just claimed)?
- [ ] Did the change survive a reality-contact test?
- [ ] Was any drift from original intent documented?

## Rationale
- Directly operationalizes the strongest discovery from the Reality Audit.
- Prevents "claimed work" from being treated as completed work.
- Aligns with external research emphasis on gated execution for self-modifying agents.
