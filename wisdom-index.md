## How to Use This File

- Add durable claims as short, testable statements with metadata.
- Prefer synthesis over accumulation: update an existing claim before adding a near-duplicate.
- Keep sources concrete: file paths, named papers/books, verified web sources, or observed system behavior.
- Use `status: working` for promising but under-verified ideas.
- Use `status: revise` when evidence conflicts or the framing has drifted.
- Promote unresolved tensions to **Updates & Contradictions** instead of hiding them.
- Keep startup lean: this file is a research substrate, not mandatory boot context.
